# Vault Agent Guide

`vault/` is the Obsidian-facing knowledge layer. It must stay clean and readable.

Only these subdirectories belong here:

- `Inbox/`
- `Reports/`
- `Notes/`
- `Dashboard.md`

`Dashboard.md` is the Obsidian-facing review surface generated by `pac dashboard build`.
It may contain Dataview queries and static summaries, but source-of-truth workflow state remains in
`library/`.

If an external frontend such as Notion is also used, Obsidian remains the readable Markdown
knowledge layer. Keep external IDs and sync ledgers out of `vault/`; expose only useful links,
frontmatter, tags, and wiki-link navigation.

Use Obsidian wiki links for readable navigation:

- Reports: `[[Reports/<object_id>|Title]]`
- Notes: `[[Notes/<object_id>|Title]]`
- Related artifacts: link explicitly when the relationship is useful to a reader.

Do not place Python code, generated indexes, caches, raw PDFs, cloned repositories, or metadata databases in this folder.
