# Inbox Agent Guide

`vault/Inbox/` is for unmanaged intake.

Use it for:

- Lists of links to ingest.
- Brief human notes about why a source may matter.
- Reading queues.

Inbox notes are unmanaged and do not appear as dashboard objects until they are ingested.

An external clipper inbox can be used as a separate capture queue. Do not mirror every clip into
`vault/Inbox/`; import only accepted items into PaC metadata and write Codex feedback in the
external inbox when that workflow is configured.

Do not write final reports or deep notes here. Once a source becomes managed, PaC metadata must live in `library/`, reports in `vault/Reports/`, deep notes in `vault/Notes/`, and dashboard visibility should come from object metadata.
